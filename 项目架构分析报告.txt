一、项目概述
--------------------------------------------------------------------------------
MusicCloud 是一个全栈音乐流媒体管理系统，支持音乐库自动扫描、在线流式播放、
歌单管理、收藏、听歌统计、天气展示等功能。采用前后端分离架构，后端提供 RESTful
API，前端为单页应用（SPA）并支持 PWA 离线缓存。

技术栈：
  后端  — Python 3.14 + FastAPI + pyodbc + SQLAlchemy ORM + APScheduler
  数据库 — Microsoft SQL Server（本地 Windows 集成验证）
  前端  — 原生 JavaScript（无框架依赖）+ HTML5 Audio + Web Audio API
  部署  — 单服务器，FastAPI 同时挂载前端静态文件和 /api 路由

三、数据库设计（Microsoft SQL Server — MusicCloudDB）
--------------------------------------------------------------------------------
共 11 张表，外键使用 ON DELETE CASCADE 保证级联删除：

 1. Users               — 用户账户与个人资料
     主键: UserID (INT, auto)
     字段: Username, PasswordHash, Nickname, AvatarUrl, Role, IsActive,
           Country, Province, City, District

 2. Songs               — 音乐文件元数据（联合唯一: Title + Artist, 独立唯一: FilePath）
     主键: SongID (INT, auto)
     字段: Title, Artist, Album, FilePath, CoverPath, Folder,
           Duration, Bitrate, SampleRate

 3. Artists             — 歌手名称（Name 唯一）
     主键: ArtistID (INT, auto)
     字段: Name

 4. Song_Artist_Mapping — 歌曲-歌手多对多映射（联合主键: SongID + ArtistID）
     字段: SongID (FK), ArtistID (FK)

 5. Playlists           — 用户歌单
     主键: PlaylistID (INT, auto)
     字段: UserID (FK), Name, CreatedAt

 6. PlaylistSongs       — 歌单-歌曲映射（联合主键: PlaylistID + SongID）
     字段: PlaylistID (FK), SongID (FK)

 7. Favorites           — 用户收藏（联合唯一: UserID + SongID）
     字段: UserID (FK), SongID (FK)

 8. PlayStats           — 用户播放次数统计（联合主键: UserID + SongID）
     字段: UserID (FK), SongID (FK), play_count, LastPlayed

 9. RefreshTokens       — JWT 刷新令牌
     主键: TokenID (INT, auto)
     字段: UserID (FK), Token, ExpiresAt

10. AuditLogs           — 操作审计日志
     主键: LogID (BIGINT, auto)
     字段: UserID, Action, IPAddress, Timestamp

11. SongLyrics          — 歌词缓存（主键: SongID）
     字段: SongID (FK), LyricsText


四、后端 API 端点一览
--------------------------------------------------------------------------------
认证：
  POST   /api/auth/login                 — 用户登录（返回 JWT access + refresh token）
  POST   /api/auth/refresh               — 刷新令牌

用户个人资料：
  GET    /api/users/me                   — 获取当前用户信息
  PUT    /api/users/me/profile           — 修改昵称
  PUT    /api/users/me/location          — 保存地区偏好（天气定位）
  POST   /api/users/me/avatar            — 上传头像
  PUT    /api/users/me/password          — 修改密码

管理功能（须 Admin 角色）：
  POST   /api/admin/users                — 创建新用户

歌单管理：
  POST   /api/playlists                  — 创建歌单
  GET    /api/playlists                  — 列出我的歌单
  POST   /api/playlists/{id}/songs       — 添加歌曲到歌单
  GET    /api/playlists/{id}/songs       — 获取歌单歌曲（含艺术家聚合）
  DELETE /api/playlists/{id}             — 删除歌单

收藏与统计：
  POST   /api/favorites/{song_id}        — 切换收藏（红心）
  GET    /api/favorites                  — 列出收藏歌曲
  POST   /api/stats/{song_id}            — 记录一次播放（≥30s 连续播放）
  GET    /api/stats/summary              — 听歌报告（Top5 歌曲 + Top5 歌手）

音乐库：
  GET    /api/songs                      — 获取所有歌曲（含艺术家聚合）
  GET    /api/stream/{song_id}           — 音频流（HTTP 206 Range 支持）
  GET    /api/song_info/{song_id}        — 音质信息（ffprobe 实时读取）
  GET    /api/lyrics/{song_id}           — 获取歌词（从网易云 API 代理）
  POST   /api/library/scan               — 手动触发音乐库扫描（Admin）

天气与定位：
  GET    /api/weather                    — 天气代理（隐藏 API Key）
  GET    /api/get-ip-info                — IP 归属地探测（支持 VPN/代理检测）
  GET    /api/regions                    — 级联行政区划数据


五、前端架构（原生 JavaScript SPA）
--------------------------------------------------------------------------------
设计模式：无框架、纯原生 JS 单页应用 + PWA

核心模块（app.js）：
  ┌─ Auth Layer        — 登录、Token 管理、自动登录检查
  ├─ Library Layer     — 歌曲文件夹浏览、搜索、筛选
  ├─ Player Core       — HTML5 Audio 播放器、音质切换、HTTP Range 请求
  ├─ Visualizer        — Web Audio API 频谱可视化（2048 点 FFT）
  ├─ Playlist Manager  — 歌单 CRUD、添加歌曲、删除歌单
  ├─ Favorites System  — 红心收藏、跨组件状态同步
  ├─ Lyrics Engine     — LRC 解析、桌面+移动端双轨同步滚动
  ├─ Location/Weather  — 四级级联地区选择 + 天气卡片
  ├─ Stats Dashboard   — Top 5 歌曲/歌手排行榜
  ├─ UI Layer          — 模态框、Toast、面板拖拽、响应式布局
  └─ PWA Layer         — Service Worker 缓存、manifest.json

视图层级（index.html）：
  - 桌面端：三栏布局（左侧封面+菜单 / 中间列表 / 右侧歌词），面板可拖拽调整宽度
  - 移动端：底部 Tab 栏切换（曲库 / 播放 / 我的），左侧菜单变为抽屉式

播放模式：列表循环 / 单曲循环 / 随机播放
音质选择：根据实际音频属性动态渲染选项（Hi-Res / 无损 / 极高 / 标准）


六、音乐扫描器设计（scanner.py）
--------------------------------------------------------------------------------
核心流程：scan_and_sync() → 遍历 MUSIC_DIR → 提取元数据 → 去重入库

元数据提取：
  - 使用 mutagen 库解析 MP3/FLAC/WAV
  - 提取：标题、专辑、时长、比特率、采样率、内嵌封面、艺术家列表
  - 封面使用 MD5(专辑名 或 标题).jpg 命名，/covers 目录存储

去重策略（三轮递进）：
  入站去重：路径级 + (Title, Artist) 复合键 → O(1) 查重
  同步后清洗：
    Pass 1 — (Title, Album, Artist) 三字段全匹配 → 保留最旧 SongID，删除其余
    Pass 2 — (Title, Artist) 完全相同 + 跨不同专辑 → 合并到一条记录 + 合并封面/专辑信息

同步后校验：
  - 文件存在性校验（清理已删除的音频文件记录）
  - 封面完整性校验（清理指向不存在文件的 CoverPath）

后台任务：APScheduler 每周日 3:00 AM 自动全量扫描


七、安全设计
--------------------------------------------------------------------------------
  - JWT 双令牌机制：access_token（3天）+ refresh_token（7天）
  - bcrypt 密码哈希存储
  - API Key 通过 .env 环境变量注入，不硬编码
  - 所有 API 端点通过 Bearer Token 鉴权（/api/weather 除外）
  - 管理员操作（创建用户、触发扫描）有角色权限检查
  - 歌单操作有所有权验证
  - 天气 API 通过后端代理，前端不暴露 API Key
  - IP 归属地探测自动识别内网/VPN/隧道（Tailscale/ZeroTier 等），改用服务器公网出口
